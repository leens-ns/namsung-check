교사·방과후강사·관리자 사용 매뉴얼

메인 화면으로공통교사방과후강사관리자비용·데이터문제 해결

공통 시작 방법

  1. Chrome에서 학교 출결관리 주소를 엽니다.
  2. Google 계정으로 로그인을 누르고 등록된 계정을 선택합니다.
  3. 상단의 알림 켜기를 눌러 Chrome 알림을 허용합니다.
  4. 공용 컴퓨터에서는 사용 후 반드시 로그아웃합니다.
출석방과후 참여 가능
결석방과후 미참여
지각교사가 등교 상태 입력
조퇴교사가 귀가 상태 입력

학생 정보와 학부모 연락처를 화면 촬영·메신저 전송·개인 기기 저장하지 마세요.

담임교사 매뉴얼

운영 요일 출결 입력

  1. 로그인 후 출결 입력에서 자신의 학급과 학생 수를 확인합니다.
  2. 대부분 출석이면 미입력 학생 모두 출석을 먼저 누릅니다.
  3. 결석·지각·조퇴 학생만 해당 버튼으로 변경합니다. 특이사항은 필요한 경우에만 입력합니다.
  4. 방과후 수강 표시가 있는 학생은 상태를 한 번 더 확인합니다.
  5. 입력 내용 재확인을 누르고 목록을 검토한 뒤 확정 저장합니다.

학생 명단 변경

  • 학생 추가에서 이름·학년·반·번호와 방과후 수강 여부를 입력합니다.
  • 수정·삭제 전 확인창의 학생 정보를 다시 확인합니다.

관리자가 지정한 방과후 운영 요일에만 입력할 수 있으며, 같은 요일에 아침 입력 알림과 교직원 재확인 알림을 받습니다. 기본 운영 요일은 월요일과 금요일입니다.

방과후강사 매뉴얼

  1. 관리자에게 등록한 이메일과 동일한 본인의 Google 계정으로 로그인합니다.
  2. 출결 조회에 자신의 담당 부서 학생만 표시되는지 확인합니다.
  3. 조회 날짜를 선택하고 최신 출결 새로고침을 누릅니다.
  4. 출석·결석과 특이사항을 확인한 뒤 수업 준비에 반영합니다. 교사가 입력한 결석·지각·조퇴는 방과후 화면에서 모두 결석으로 표시되며 상단 결석 수에도 합산됩니다.

이메일 주소를 아는 것만으로는 접속할 수 없습니다. Google 로그인을 통과하고 등록 이메일이 정확히 일치해야 합니다. 별도 학교 계정을 만들 필요 없이 강사 개인 Google 계정을 등록할 수 있습니다.

출결 입력·학생 수정·학부모 연락처 조회 권한은 없습니다. 관리자가 지정한 운영 요일과 시간에 방과후 출결 확인 알림만 받습니다.

관리자 매뉴얼

학기 시작 설정

  1. 관리자 설정에서 CSV 학생 명단을 가져옵니다.
  2. 방과후 운영 요일과 학년별 최대 반 수를 확인합니다. 기본값은 월·금, 학년별 3개 반입니다.
  3. 담임교사 이메일과 학급을 일괄 배정합니다.
  4. 방과후 부서 목록을 월요일·금요일로 나누어 정리합니다.
  5. 방과후강사 CSV로 이메일과 담당 부서를 등록합니다.
  6. 추가 관리자를 지정합니다. 관리자도 담임 학급을 함께 배정할 수 있습니다.

관리자 담임 학급

  • 담임교사 학급 배정에서 관리자 이메일에도 담당 학급을 배정할 수 있습니다.
  • 관리자에게 담임 학급을 배정하거나 변경하면 출결 입력 화면이 즉시 자신의 반으로 바뀌며, 다음 로그인 때도 그 반이 먼저 표시됩니다.
  • 관리자 권한은 그대로 유지되므로 학급 필터에서 전체 또는 다른 반으로 바꾸어 확인할 수 있습니다.

알림 설정

  • 아침 입력 알림은 관리자·담임교사 대상입니다.
  • 교직원 재확인 알림과 방과후강사 확인 알림 시간을 각각 설정합니다.
  • 세 알림 모두 선택한 운영 요일에만 발송됩니다. 기본 확인 시간은 교직원 14:05, 방과후강사 14:10입니다.

개인정보와 계정

  • 연락처가 꼭 필요할 때만 학부모 연락처 비공개 체크를 해제하고, 확인 후 다시 체크합니다.
  • 연락처 설정은 관리자 설정에만 있으며 방과후강사 화면에는 표시되지 않습니다.
  • 강사 이메일 등록은 조회 권한의 허용 목록입니다. 강사는 반드시 같은 이메일의 Google 계정으로 로그인해야 합니다.
  • 계정 퇴직·담당 변경 시 즉시 권한을 삭제하거나 새 부서로 수정합니다.

학생 CSV는 공개 Google 문서 링크로 연결하지 말고 관리자 기기의 파일로만 업로드하세요.

비용과 데이터 관리

무료 운영 기준

  • 현재 시스템은 Firebase의 무료 Spark 요금제 범위에서 운영하도록 구성되어 있습니다.
  • Firestore 무료 한도는 하루 읽기 50,000건, 쓰기 20,000건, 삭제 20,000건이며 저장 공간은 1GiB입니다.
  • Google 로그인은 일반 인증 기준 월간 활성 사용자 50,000명까지 무료이고, 웹 푸시 알림인 Firebase Cloud Messaging은 무료입니다.
  • GitHub 저장소가 공개 상태이고 표준 실행기를 사용하면 현재 알림 예약 작업의 GitHub Actions 실행 비용은 없습니다.

학교 예상 사용량

교직원·강사 약 50명, 학생 약 450명, 주 2회 운영을 가정하면 일반적인 하루 읽기는 대략 5,000~15,000건, 출결 쓰기는 운영일 기준 약 450건 수준입니다. 여러 관리자가 반복 조회하는 상황을 포함해도 무료 한도보다 충분히 낮습니다.

같은 출결을 다시 확정해도 변경된 학생만 저장합니다. 최신 출결 새로고침은 10초 간격으로 제한되며, 같은 기기의 알림 토큰은 하루 한 번만 갱신됩니다.

관리자가 확인할 항목

  1. Firebase Console의 Firestore Database → 사용량에서 읽기·쓰기·저장 공간을 월 1회 확인합니다.
  2. 결제 계정을 연결하기 전에는 무료 한도를 넘겨도 자동 과금되지 않고 해당 서비스 사용이 제한됩니다.
  3. 출결 자료의 법적·학교 내부 보관 기간을 정한 뒤, 필요 기간이 지난 자료는 별도 보관 또는 삭제 방침에 따라 정리합니다. 자동 유료 백업·PITR·TTL 기능은 사용하지 않습니다.

비용 경고를 피하려고 개인정보를 공개 저장소나 개인 파일로 옮기면 안 됩니다. 학생 CSV와 실제 출결 데이터는 GitHub에 게시되지 않고 Firestore에만 저장됩니다.

문제 해결

  • 로그인 불가: 등록된 이메일인지 관리자에게 확인하고 Chrome에서 올바른 Google 계정을 선택합니다.
  • 체험판에서 바로 로그인됨: 체험판의 역할 버튼은 화면 확인을 위해 인증을 생략합니다. 실제 배포 앱에는 적용되지 않습니다.
  • 입력 버튼이 비활성화됨: 오늘이 관리자 설정의 방과후 운영 요일인지 확인합니다.
  • 학생이 안 보임: 담임 학급 또는 강사 담당 부서 배정을 확인합니다.
  • 최신 출결이 안 보임: 조회 날짜를 확인하고 최신 출결 새로고침을 누릅니다.
  • 알림이 안 옴: Chrome 사이트 설정에서 알림 허용 여부와 운영체제 집중 모드를 확인합니다.
  • GitHub 실패 메일이 반복됨: 맥북 상태와 무관한 서버 인증 오류입니다. 예약 작업은 실패 메일을 반복 발송하지 않고 해당 회차 알림만 건너뛰도록 구성되어 있습니다. 관리자는 Google Cloud 알림 연결을 복구한 뒤 작업을 수동 실행해 확인합니다.
  • 연락처가 안 보임: 관리자만 조회할 수 있습니다. 관리자 설정에서 학부모 연락처 비공개 체크를 해제했는지 확인합니다.
