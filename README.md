# 남성초등학교 출결관리 시스템 배포 설정

Firebase 설정 전에는 `demo.html`을 열어 관리자·교사·방과후강사 화면과 출결 기능을 체험할 수 있습니다. 체험판 데이터는 해당 브라우저에만 저장되며 실제 운영에는 사용하지 않습니다.

1. Firebase 프로젝트를 만들고 Authentication에서 Google 로그인을 활성화합니다.
2. Firestore 데이터베이스를 만든 뒤 `firestore.rules` 내용을 보안 규칙으로 배포합니다.
3. Firebase 웹 앱 설정값을 `config.js`의 `firebase` 항목에 입력합니다.
4. Firebase Hosting 또는 HTTPS 웹 서버에 이 폴더를 배포합니다.
5. Authentication의 승인된 도메인에 실제 배포 도메인을 등록합니다.

Firebase CLI를 사용하는 경우 이 폴더에서 프로젝트를 연결한 뒤 Hosting, Firestore 규칙, 색인을 함께 배포하면 됩니다. 로컬 파일(`file://`)에서는 Google 로그인이 작동하지 않으므로 반드시 HTTPS 배포 주소 또는 승인된 `localhost` 주소로 실행해야 합니다.

관리자 계정은 `leens@nsworld.net`으로 보안 규칙에 고정되어 있습니다. 관리자 로그인 후 기기에 저장된 CSV 명단 파일을 선택하고 방과후강사 계정과 부서를 등록합니다. 학생 개인정보가 포함된 Google 문서를 웹에 공개하거나 공개 CSV 주소로 연결하지 마세요.

방과후강사 등록은 로그인 허용 목록을 만드는 과정입니다. 이메일 등록만으로 접속할 수 없으며, 강사가 등록 이메일과 동일한 Google 계정으로 Firebase 인증을 통과해야 합니다. 강사는 자신의 담당 부서 출결만 읽을 수 있고 학부모 연락처는 읽을 수 없습니다.

앱을 닫은 상태에서도 알림을 보내기 위해 유료 Cloud Functions 대신 공개 GitHub 저장소의 무료 Actions 예약 작업을 사용합니다. `.github/workflows/send-attendance-reminders.yml`은 5분마다 시간을 확인하고 `automation/send-reminders.mjs`가 FCM 웹 푸시를 발송합니다. Google Cloud 연결은 장기 인증키를 저장하지 않는 Workload Identity Federation 방식으로 설정해야 합니다.

앱은 방과후 운영일의 출석·결석 확인에 집중하며 월별 통계와 인쇄 기능은 사용하지 않습니다. 교사가 입력한 지각·조퇴는 원본 기록에는 유지되지만 방과후강사 조회 화면에서는 결석으로 표시됩니다.

알림 대상과 시간은 역할별로 분리됩니다. 아침 입력 알림과 교직원 재확인 알림은 관리자와 배정된 담임교사에게, 방과후강사 확인 알림은 강사에게 발송됩니다. 세 알림 모두 관리자가 선택한 운영 요일에만 발송되며 기본 요일은 월·금입니다. 기본 확인 시간은 교직원 14시 5분, 방과후강사 14시 10분입니다. 발송 시 현재 `access` 권한을 다시 확인하므로 배정 또는 권한이 삭제된 계정은 제외됩니다.

담임 배정 목록은 기본적으로 학년별 3개 반까지만 표시하며 관리자가 1~10반 사이에서 변경할 수 있습니다. 관리자 계정은 담임 학급 정보를 함께 가질 수 있으며 관리자 권한을 변경해도 학급 배정을 보존합니다.

담임 학급이 배정된 관리자는 로그인 직후 자신의 반이 출결 입력 화면의 기본 학급으로 선택됩니다. 관리자 권한과 전체 학급 필터는 유지됩니다.

사용 매뉴얼은 앱 메뉴의 `사용 매뉴얼`에서 열 수 있으며 `manual.html`을 직접 열어도 됩니다.

## 비용 및 사용량 최적화

이 앱은 Firebase Spark 무료 한도를 기준으로 운영합니다. Firestore 무료 한도는 읽기 50,000건/일, 쓰기 20,000건/일, 삭제 20,000건/일, 저장 공간 1GiB입니다. FCM 웹 푸시는 무료입니다.

- 출결 확정 저장은 신규 또는 수정된 학생 기록만 씁니다.
- 동일 기기의 FCM 토큰 문서는 하루 한 번만 갱신합니다.
- 방과후 조회 새로고침은 10초 간격으로 제한합니다.
- 알림 예약 작업은 10분 간격으로 설정 조회만 수행하고, 발송 시간이 되었을 때만 토큰과 권한을 조회합니다.
- 조회하지 않는 개인정보·메모 필드는 Firestore 단일 필드 색인을 만들지 않습니다.
- 자동 유료 백업, PITR, TTL은 사용하지 않습니다.

## 무료 자동 보관 관리

관리자는 앱의 `관리자 설정 → 무료 운영·자료 보관`에서 출결 보관 기간을 12·24·36·60개월 중 선택할 수 있습니다. 자동 삭제를 켜면 `.github/workflows/cleanup-old-attendance.yml`이 매월 2일 오전 3시 20분(한국 시간)에 실행되어 기준일 이전의 `attendance` 문서만 삭제합니다.

Firebase의 TTL 삭제는 결제 활성화가 필요한 기능이므로 사용하지 않습니다. 대신 공개 저장소의 표준 GitHub-hosted runner에서 동작하는 GitHub Actions와 Firestore 무료 삭제 할당량을 사용합니다. 한 번의 실행은 최대 10,000건으로 제한해 Firestore의 일일 무료 삭제 20,000건 아래에 머물도록 했습니다. 학생 명단(`students`)과 학부모 연락처(`contacts`)는 자동 삭제 대상이 아닙니다.

관리자 화면은 Firebase 사용량, GitHub 자동화 실행 내역, 마지막 정리 결과로 바로 이동할 수 있습니다. 정리 결과에는 날짜·삭제 건수만 기록되며 개인정보는 포함하지 않고 관리자 화면에만 표시합니다.

GitHub는 공개 저장소에 60일 동안 저장소 활동이 없으면 예약 작업을 자동으로 비활성화할 수 있습니다. 관리자는 월 1회 `배포·알림 전체 상태`를 확인하고 비활성화된 작업이 있으면 GitHub의 `Enable workflow`로 다시 켭니다.

교직원·강사 약 50명, 학생 약 450명, 주 2회 운영에서는 일반적으로 무료 한도 안에 충분히 들어옵니다. 관리자는 Firebase Console의 Firestore 사용량 화면을 월 1회 확인합니다.
